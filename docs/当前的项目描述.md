## 1. 项目概述

本项目是一个四工程并行的在线德州扑克系统，支持多人实时牌局、AI 决策、移动端与后台管理。

当前仓库以“多 AI 并行开发”为主要协作场景，核心目标是：
- **隔离性**：不同功能在不同目录开发，避免冲突。
- **局部性**：开发时只需读取本任务目录。
- **可导航性**：通过结构文档和任务描述快速定位工作区。

---

## 2. 四工程架构（当前真实）

```text
dezhou/
├─ admin/     # 后台前端
├─ mobile/    # 移动端前端
├─ nodejs/    # 主业务后端 + Socket.IO
├─ python/    # AI 决策服务
└─ docs/      # 协作文档
```

调用关系：
- `admin -> nodejs`（HTTP）
- `mobile -> nodejs`（HTTP + Socket.IO）
- `nodejs -> python`（HTTP）

## 3. 当前代码结构状态

### 3.1 Admin

已形成主分层：
- `admin/src/app`：入口与路由装配
- `admin/src/layouts/admin-shell`：后台壳层
- `admin/src/shared`：共享 API / 样式
- `admin/src/features`：dashboard / room / player / record
- `admin/src/pages`：页面装配层

现状说明：
- `views/api/router/assets` 等历史目录仍在，属于过渡存量。

### 3.2 Mobile

已形成主分层：
- `mobile/src/app`：入口与路由装配
- `mobile/src/layouts/mobile-shell`：移动壳层
- `mobile/src/shared`：API / socket / utils / styles
- `mobile/src/pages`：已建立 7 个页面入口

现状说明：
- 目前 `pages` 仍对 `views` 做转发，`views/stores/components` 仍承载大量业务逻辑。

### 3.3 Nodejs

已形成“新装配 + 存量业务”结构：
- `nodejs/src/app`：HTTP/Socket 装配与注册
- `nodejs/src/config`：环境变量与常量配置
- `nodejs/src-server`：认证、大厅、房间、游戏、后台等存量业务模块

当前安全基线（已落地）：
- 管理端鉴权中间件（可通过 `ADMIN_AUTH_ENABLED=true` 启用）
- `/api/*` 与 `/api/auth/*` 速率限制
- CORS 白名单（`CORS_ORIGINS`）
- 生产环境关键 env 校验（`JWT_SECRET`、`MONGODB_URI`）

### 3.4 Python

已形成基础分层：
- `python/src/api`：FastAPI 路由与 schema
- `python/src/services`：决策流程
- `python/src/domain`：牌力规则
- `python/src/infrastructure`：第三方库适配

兼容入口：
- `python/api.py` 保留 `api:app`
- `python/app.py` 负责进程启动

## 5. 协作开发约束（必须遵守）

1. 一个任务只能绑定一个工作根目录。
2. 禁止跨工程混改（`admin/mobile/nodejs/python` 不能在同任务同时修改）。
3. 共享层改动必须单独任务处理（如 `shared/config/docs`）。
4. 新代码优先进入已定义层级目录，不新增无职责散目录。

任务分配建议（简版）：
- Admin 页面任务：`admin/src/pages` + `admin/src/features`
- Mobile 页面任务：`mobile/src/pages`（必要时联动 `mobile/src/views`）
- Node 业务任务：`nodejs/src-server/<单模块>`
- Python 决策任务：`python/src/services` + `python/src/domain`

## 6. 运行与构建

### 6.1 Admin
- 开发：`npm run start`（在 `admin` 目录）
- 构建：`npx vite build`

### 6.2 Mobile
- 开发：`npm run start`（在 `mobile` 目录）
- 构建：`npx vite build`
- APK：`npm run apk`

### 6.3 Nodejs
- 开发：`npm run dev`（在 `nodejs` 目录）
- 运行：`npm run start`

### 6.4 Python
- 运行：`python app.py`（在 `python` 目录）
