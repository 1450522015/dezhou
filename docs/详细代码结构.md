# 详细代码结构（AI 并行协作版）

> 版本日期：2026-04-10  
> 适用范围：`dezhou` 当前仓库  
> 文档目标：让多个 AI 在同一仓库并行开发时做到 **隔离性、局部性、可导航性**。

---

## 1. 核心原则

### 1.1 隔离性（避免冲突）

1. 一个任务只能绑定一个“工作根目录”（Working Root）。
2. AI 只能改自己工作根目录内的文件。
3. 跨目录改动必须拆成独立任务（先共享层，再业务层）。
4. 禁止在同一任务里同时改多个工程（`admin/mobile/nodejs/python`）。

### 1.2 局部性（减少扫描）

1. 每个任务必须声明“允许读取目录”和“禁止读取目录”。
2. 默认只读本任务目录；只有排障时才临时扩展读取范围。
3. 新功能优先落在既有分层目录，不在根层随意新建散目录。

### 1.3 可导航性（快速定位）

1. 任务描述必须包含：工程名 + 功能域 + 目标目录。
2. 每个目录必须有明确职责，不允许“什么都能放”。
3. 本文档中的“任务路由表”是 AI 的首选导航入口。

## 3. 工程级边界（按当前代码）

### 3.1 Admin（后台前端）

当前重点目录：

```text
admin/src/
├─ app/                    # 应用装配层（入口、路由）
├─ layouts/                # 布局层（admin-shell）
├─ shared/                 # 共享层（api、styles）
├─ features/               # 业务功能层（dashboard/room/player/record）
├─ pages/                  # 页面装配层
├─ views/                  # 历史页面目录（存量）
├─ api/ router/ assets/    # 历史/过渡目录
```

边界规则：
- 页面开发任务：只改 `admin/src/pages/*` + 对应 `admin/src/features/*`。
- 后台接口适配任务：只改 `admin/src/shared/api/*`。
- 样式规范任务：只改 `admin/src/shared/styles/*`。
- 禁止在 `admin` 任务中修改 `mobile/nodejs/python`。

---

### 3.2 Mobile（移动前端）

当前重点目录：

```text
mobile/src/
├─ app/                    # 应用装配层（已启用）
├─ layouts/                # 布局层（mobile-shell）
├─ pages/                  # 页面入口层（已建 7 个 page）
├─ shared/                 # 共享层（api/socket/utils/styles）
├─ views/                  # 存量页面实现（当前由 pages 转发）
├─ stores/                 # 全局状态（存量）
├─ components/             # 组件（存量）
├─ api/socket/utils/router # 历史/兼容目录
```

边界规则：
- 页面任务：优先改 `mobile/src/pages/*`，必要时改对应 `mobile/src/views/*`。
- 网络层任务：只改 `mobile/src/shared/api/*`、`mobile/src/shared/socket/*`。
- 状态重构任务：只改 `mobile/src/stores/*` 或新建 `mobile/src/features/*`（需单独任务）。
- 禁止在 `mobile` 任务中修改 `admin/nodejs/python`。

---

### 3.3 Nodejs（主后端）

当前重点目录：

```text
nodejs/
├─ src/
│  ├─ app/                 # 新装配层（http/socket/register）
│  └─ config/              # 新配置层（env/constants）
├─ src-server/             # 存量业务实现（auth/game/lobby/...）
├─ index.js                # 启动入口
```

边界规则：
- 装配/启动任务：只改 `nodejs/src/app/*`、`nodejs/src/config/*`。
- 业务接口任务：只改 `nodejs/src-server/<模块>.js`（单模块单任务）。
- 大重构任务（modules/domain 化）必须拆分：
  1) 新目录与接口壳  
  2) 单模块迁移  
  3) 删除兼容层

当前已落地（安全基线）：
- 管理接口鉴权：`nodejs/src-server/adminAuth.js` + `admin.js` 路由级保护（`ADMIN_AUTH_ENABLED=true` 时启用，默认兼容关闭）。
- 速率限制：`nodejs/src/app/http-server.js` 对 `/api/*` 与 `/api/auth/*` 启用限流。
- CORS 白名单：由 `CORS_ORIGINS` 控制，配置位于 `nodejs/src/config/env.js`。
- 生产环境必需配置校验：`JWT_SECRET`、`MONGODB_URI`。

---

### 3.4 Python（AI 服务）

当前重点目录：

```text
python/
├─ src/
│  ├─ api/                 # FastAPI 路由与 schema
│  ├─ services/            # 决策服务
│  ├─ domain/              # 规则域
│  └─ infrastructure/      # 第三方库适配
├─ api.py                  # 兼容导出 api:app
├─ app.py                  # 进程启动
```

边界规则：
- 接口任务：只改 `python/src/api/*`。
- 决策策略任务：只改 `python/src/services/*`、`python/src/domain/*`。
- 第三方依赖封装任务：只改 `python/src/infrastructure/*`。
- 禁止在 `python` 任务中修改前端目录。

## 4. AI 任务路由表（导航入口）

> 用法：先按“任务类型”定位，再按“工作根目录”执行。

| 任务类型 | 工作根目录（只读写这里） | 允许联动目录 | 禁止修改 |
|---|---|---|---|
| Admin 页面改版 | `admin/src/pages` | `admin/src/features` | `mobile/nodejs/python` |
| Admin 接口适配 | `admin/src/shared/api` | `admin/src/features` | `mobile/nodejs/python` |
| Mobile 页面改版 | `mobile/src/pages` | `mobile/src/views` | `admin/nodejs/python` |
| Mobile 网络通信 | `mobile/src/shared/api`、`mobile/src/shared/socket` | `mobile/src/stores` | `admin/nodejs/python` |
| Nodejs 启动/装配 | `nodejs/src/app`、`nodejs/src/config` | `nodejs/index.js` | `admin/mobile/python` |
| Nodejs 业务模块 | `nodejs/src-server/<单文件>` | 同模块必要依赖 | `admin/mobile/python` |
| Python API 层 | `python/src/api` | `python/src/services` | `admin/mobile/nodejs` |
| Python 决策逻辑 | `python/src/services`、`python/src/domain` | `python/src/infrastructure` | `admin/mobile/nodejs` |
| 文档治理 | `docs` | 无 | 业务代码目录 |

## 5. 依赖方向约束（当前执行版）

### 5.1 Admin

```text
app -> layouts -> pages -> features -> shared
```

约束：
- `pages` 不直接请求底层 HTTP，统一走 `features` / `shared/api`。
- `shared` 不反向依赖 `features/pages`。

### 5.2 Mobile

```text
app -> layouts -> pages -> (views 过渡) -> shared/stores
```

约束：
- 新逻辑优先写入 `pages/shared`，旧逻辑可暂留 `views/stores`。
- 新增通信逻辑统一放 `shared/api` 或 `shared/socket`。

### 5.3 Nodejs

```text
index -> src/app -> src-server(存量业务)
```

约束：
- `src/app` 只做装配，不写业务规则。
- 业务继续在 `src-server` 分模块维护，避免单文件混改。

### 5.4 Python

```text
api -> services -> domain
services -> infrastructure
```

约束：
- `api` 不写复杂决策。
- 第三方扑克库仅通过 `infrastructure` 使用。

## 6. 并行开发防冲突规则（强制）

1. 同一时刻，一个目录只能分配给一个 AI 任务。
2. 任务开始前先声明：
   - 工作根目录
   - 只读目录
   - 禁改目录
3. 若需要改共享层（`shared/config/docs`），必须单独开任务并先合并。
4. 禁止“顺手修复”非任务目录代码。
5. 提交说明中必须包含“改动目录清单”。
